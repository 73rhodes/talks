"Concurrency - a bug story" might not be the best title because concurrency in the traditional sense is not really a thing in NodeJS. It is commonly believed that "Javascript is single-threaded" and we don't have concurrency in the same way we as we do in a real-time, multi-threaded C++ application. However as we will see this is not excactly the right way to look at it, and there are some very interesting comparisons between realtime systems concepts, like concurrency, and asynchronous programming. So this is really a talk about the parallels between multi-threaded realtime systems and asynchronous event-driven programming in NodeJS and what lessons we can learn from them. So the bug story is a starting point to talk about real- time, multi-threaded systems concepts as they apply to an asyncronous programming environment like NodeJS.
